# 루틴 트래커 (Routine Tracker) 기획안

> 매일의 작은 습관을 체크하고, AI에게 개선 조언을 받는 개인용 습관 관리 앱

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | 루틴 트래커 (Routine Tracker) |
| 플랫폼 | iOS / Android (React Native + Expo) |
| 타깃 | 매일 습관을 꾸준히 관리하고 싶은 개인 사용자 |
| 핵심 가치 | ① 빠른 체크 ② 스트릭 동기부여 ③ AI 맞춤 조언 |
| 서버 | 없음 (완전 로컬, AsyncStorage) |
| 외부 연동 | Anthropic Claude API (AI 조언 1곳만) |

---

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| Framework | React Native (Expo SDK) | `expo` 관리형 워크플로우 |
| 언어 | TypeScript | 전 영역 타입 적용 |
| 네비게이션 | Expo Router (파일 기반 탭) | 또는 React Navigation Bottom Tabs |
| 로컬 DB | AsyncStorage | 서버 불필요, 단순 key-value |
| 스타일 | NativeWind (Tailwind CSS) | 다크모드 `dark:` 변형 활용 |
| AI | `@anthropic-ai/sdk` | 모델: `claude-haiku-4-5` (빠르고 저렴) |
| 차트 | react-native-svg 기반 커스텀 막대 | 의존성 최소화 |
| 환경변수 | `.env` + `app.config` extra | 키 외부 노출 방지 |

> **모델 선택 근거**: 조언 텍스트 생성은 짧고 비용 민감 → `claude-haiku-4-5`(빠르고 저렴)를 기본값으로. 품질을 더 원하면 `claude-sonnet-4-6`로 교체 가능. *(실제 구현 단계에서 claude-api 스킬로 최신 모델/파라미터 재확인 예정)*

## 3. 데이터 구조

```ts
// 루틴 정의
type Routine = {
  id: string;            // uuid
  title: string;         // "운동"
  category: Category;    // "건강" | "학습" | "자기계발" | "생활"
  icon: string;          // 이모지 "💪"
  createdAt: string;     // ISO 날짜
  archived?: boolean;    // 삭제 대신 보관(통계 보존용)
};

// 일별 체크 기록
type Record = {
  routineId: string;
  date: string;          // "YYYY-MM-DD"
  completed: boolean;
};

type Category = "건강" | "학습" | "자기계발" | "생활";
```

### AsyncStorage 키 설계
| 키 | 값 | 설명 |
|----|----|------|
| `@routines` | `Routine[]` | 루틴 목록 |
| `@records` | `Record[]` | 전체 체크 기록 |
| `@settings` | `{ theme, seeded }` | 테마, 샘플 생성 여부 |

> 기록은 (routineId + date) 조합으로 유일. 토글 시 해당 레코드 upsert.

## 4. 화면 구성 (4개 탭)

### 탭 1. 홈 — 오늘 루틴 체크
- 상단: 오늘 날짜(예: `6월 16일 월요일`) + 완료율 `3/5 완료` + 원형/바 진행률
- 루틴 리스트: 아이콘 · 제목 · 스트릭(🔥 N) · 체크박스
- 탭하면 완료/미완료 토글 (즉시 저장)
- **전체 완료 시 축하 애니메이션** (Confetti / Lottie 또는 스케일+페이드)

### 탭 2. 루틴 관리 — 추가/편집/삭제
- 루틴 목록 (편집 모드)
- `+` 추가: 제목 입력 · 카테고리 선택 · 아이콘(이모지) 선택
- 항목 스와이프 또는 버튼으로 수정/삭제
- 삭제 시 기록 보존 여부 확인(아카이브 권장)

### 탭 3. 통계
- **주간 달성률 막대 그래프** (최근 7일, 일별 완료 비율)
- **루틴별 달성률** (가로 바, 최근 7일 또는 30일 기준)
- **하이라이트 카드**: 가장 잘 지킨 루틴 🏆 / 가장 놓친 루틴 ⚠️

### 탭 4. AI 조언
- "AI 조언 받기" 버튼
- 누르면 최근 7일 기록을 요약 → Claude API 호출
- **호출 중 로딩 스피너** 표시
- 응답을 카드로 표시 (재요청 가능)
- 에러/키 미설정 시 안내 메시지

## 5. AI 조언 동작 흐름

```
[버튼 클릭]
   ↓
최근 7일 기록 집계
  - 루틴별 달성 횟수/비율
  - 전체 완료율 추세
   ↓
프롬프트 구성 (한국어, 데이터 요약 + 조언 요청)
   ↓
Claude API 호출 (로딩 스피너)
   ↓
조언 텍스트 렌더링
```

**프롬프트 설계 방향**
- system: "너는 친근한 습관 코치다. 한국어로 2~3문장, 구체적이고 실천 가능한 조언."
- user: 루틴별 달성률 데이터(JSON/표) 전달
- 예시 출력: *"운동 루틴 달성률이 40%네요. 시간대를 아침으로 바꿔보는 건 어떨까요?"*

**보안**
- API 키는 `.env` → `app.config.ts`의 `extra`로 주입, 코드에 하드코딩 금지
- `.env`는 `.gitignore`, `.env.example`만 커밋

> ⚠️ 참고: Expo 앱에서 클라이언트가 직접 Claude API를 호출하면 빌드된 앱에 키가 포함될 수 있음. 학습/개인용 프로토타입 전제로 진행하며, 배포 시에는 프록시 서버 경유를 별도 권장.

## 8. 핵심 로직 메모

- **스트릭 계산**: 오늘부터 거꾸로 연속 `completed=true`인 날 수. 오늘 미체크면 어제까지로 계산.
- **완료율**: (오늘 완료 루틴 수) / (활성 루틴 수).
- **주간 그래프**: 최근 7일 각 날짜의 `완료/활성` 비율.
- **삭제 정책**: archive 우선 → 통계 왜곡 방지.

## 10. 배포 전략 & 단계별 로드맵

> **목표**: 플레이스토어 **공개 배포**. 단, 한 번에 가지 않고 단계로 나눠 진행.

### 🟢 1차 (v1) — 현재 기획안 범위, 빠른 구현
- 본 문서의 4탭 기능 전부 구현 (홈 / 루틴 관리 / 통계 / AI 조언)
- AI 조언: **클라이언트에서 Claude API 직접 호출** (`.env` 키)
- 목적: 기능 완성 + 내부 테스트(개발 빌드 / 내부 테스트 트랙)
- ⚠️ 이 단계 빌드는 **공개 스토어에 올리지 않음** (키 유출 위험 때문)

### 🟡 2차 (확장) — 공개 배포 준비
- 기능 확장/다듬기 (추후 정의)
- **[공개 배포 필수 차단 항목] AI 호출을 프록시 서버 경유로 전환**
  - 서버리스 함수(예: Cloudflare Workers / Vercel / Supabase Edge) 뒤로 키 숨김
  - 앱은 프록시 엔드포인트만 호출, 키는 서버 환경변수에만 존재
- 스토어 자산: 아이콘/스플래시, 스크린샷, 개인정보처리방침, `android.package`/`versionCode`

### 🔴 3차 — 플레이스토어 공개 배포
- `eas build -p android` (AAB) → 내부 테스트 → 비공개 테스트 → 프로덕션
- **키 프록시 전환 완료가 공개 배포의 전제 조건** (미완료 시 배포 보류)
