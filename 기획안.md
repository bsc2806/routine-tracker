# 루틴 트래커 (Routine Tracker) 기획안

> 매일의 작은 습관을 체크하고, AI에게 개선 조언을 받는 개인용 습관 관리 앱

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | 루틴 트래커 (Routine Tracker) |
| 플랫폼 | iOS / Android (React Native + Expo) |
| 타깃 | 매일 습관을 꾸준히 관리하고 싶은 개인 사용자 |
| 핵심 가치 | ① 빠른 체크 ② 스트릭 동기부여 ③ AI 맞춤 조언 |
| 서버 | 없음 (완전 로컬, AsyncStorage) |
| 외부 연동 | Anthropic Claude API (AI 조언 1곳만) |

---

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| Framework | React Native (Expo SDK) | `expo` 관리형 워크플로우 |
| 언어 | TypeScript | 전 영역 타입 적용 |
| 네비게이션 | Expo Router (파일 기반 탭) | 또는 React Navigation Bottom Tabs |
| 로컬 DB | AsyncStorage | 서버 불필요, 단순 key-value |
| 스타일 | NativeWind (Tailwind CSS) | 다크모드 `dark:` 변형 활용 |
| AI | `@anthropic-ai/sdk` | 모델: `claude-haiku-4-5` (빠르고 저렴) |
| 차트 | react-native-svg 기반 커스텀 막대 | 의존성 최소화 |
| 환경변수 | `.env` + `app.config` extra | 키 외부 노출 방지 |

> **모델 선택 근거**: 조언 텍스트 생성은 짧고 비용 민감 → `claude-haiku-4-5`(빠르고 저렴)를 기본값으로. 품질을 더 원하면 `claude-sonnet-4-6`로 교체 가능. *(실제 구현 단계에서 claude-api 스킬로 최신 모델/파라미터 재확인 예정)*

## 3. 데이터 구조

```ts
// 루틴 정의
type Routine = {
  id: string;            // uuid
  title: string;         // "운동"
  category: Category;    // "건강" | "학습" | "자기계발" | "생활"
  icon: string;          // 이모지 "💪"
  createdAt: string;     // ISO 날짜
  archived?: boolean;    // 삭제 대신 보관(통계 보존용)
};

// 일별 체크 기록
type Record = {
  routineId: string;
  date: string;          // "YYYY-MM-DD"
  completed: boolean;
};

type Category = "건강" | "학습" | "자기계발" | "생활";
```

### AsyncStorage 키 설계
| 키 | 값 | 설명 |
|----|----|------|
| `@routines` | `Routine[]` | 루틴 목록 |
| `@records` | `Record[]` | 전체 체크 기록 |
| `@settings` | `{ theme, seeded }` | 테마, 샘플 생성 여부 |

> 기록은 (routineId + date) 조합으로 유일. 토글 시 해당 레코드 upsert.

## 4. 화면 구성 (4개 탭)

### 탭 1. 홈 — 오늘 루틴 체크
- 상단: 오늘 날짜(예: `6월 16일 월요일`) + 완료율 `3/5 완료` + 원형/바 진행률
- 루틴 리스트: 아이콘 · 제목 · 스트릭(🔥 N) · 체크박스
- 탭하면 완료/미완료 토글 (즉시 저장)
- **전체 완료 시 축하 애니메이션** (Confetti / Lottie 또는 스케일+페이드)

### 탭 2. 루틴 관리 — 추가/편집/삭제
- 루틴 목록 (편집 모드)
- `+` 추가: 제목 입력 · 카테고리 선택 · 아이콘(이모지) 선택
- 항목 스와이프 또는 버튼으로 수정/삭제
- 삭제 시 기록 보존 여부 확인(아카이브 권장)

### 탭 3. 통계
- **주간 달성률 막대 그래프** (최근 7일, 일별 완료 비율)
- **루틴별 달성률** (가로 바, 최근 7일 또는 30일 기준)
- **하이라이트 카드**: 가장 잘 지킨 루틴 🏆 / 가장 놓친 루틴 ⚠️

### 탭 4. AI 조언
- "AI 조언 받기" 버튼
- 누르면 최근 7일 기록을 요약 → Claude API 호출
- **호출 중 로딩 스피너** 표시
- 응답을 카드로 표시 (재요청 가능)
- 에러/키 미설정 시 안내 메시지

## 5. AI 조언 동작 흐름

```
[버튼 클릭]
   ↓
최근 7일 기록 집계
  - 루틴별 달성 횟수/비율
  - 전체 완료율 추세
   ↓
프롬프트 구성 (한국어, 데이터 요약 + 조언 요청)
   ↓
Claude API 호출 (로딩 스피너)
   ↓
조언 텍스트 렌더링
```

**프롬프트 설계 방향**
- system: "너는 친근한 습관 코치다. 한국어로 2~3문장, 구체적이고 실천 가능한 조언."
- user: 루틴별 달성률 데이터(JSON/표) 전달
- 예시 출력: *"운동 루틴 달성률이 40%네요. 시간대를 아침으로 바꿔보는 건 어떨까요?"*

**보안**
- API 키는 `.env` → `app.config.ts`의 `extra`로 주입, 코드에 하드코딩 금지
- `.env`는 `.gitignore`, `.env.example`만 커밋

> ⚠️ 참고: Expo 앱에서 클라이언트가 직접 Claude API를 호출하면 빌드된 앱에 키가 포함될 수 있음. 학습/개인용 프로토타입 전제로 진행하며, 배포 시에는 프록시 서버 경유를 별도 권장.

## 8. 핵심 로직 메모

- **스트릭 계산**: 오늘부터 거꾸로 연속 `completed=true`인 날 수. 오늘 미체크면 어제까지로 계산.
- **완료율**: (오늘 완료 루틴 수) / (활성 루틴 수).
- **주간 그래프**: 최근 7일 각 날짜의 `완료/활성` 비율.
- **삭제 정책**: archive 우선 → 통계 왜곡 방지.

## 10. 배포 전략 & 단계별 로드맵

> **목표**: 플레이스토어 **공개 배포**. 단, 한 번에 가지 않고 단계로 나눠 진행.

### 🟢 1차 (v1) — 현재 기획안 범위, 빠른 구현
- 본 문서의 4탭 기능 전부 구현 (홈 / 루틴 관리 / 통계 / AI 조언)
- AI 조언: **클라이언트에서 Claude API 직접 호출** (`.env` 키)
- 목적: 기능 완성 + 내부 테스트(개발 빌드 / 내부 테스트 트랙)
- ⚠️ 이 단계 빌드는 **공개 스토어에 올리지 않음** (키 유출 위험 때문)

### 🟡 2차 (확장) — 공개 배포 준비
- 기능 확장/다듬기 (추후 정의)
- **[공개 배포 필수 차단 항목] AI 호출을 프록시 서버 경유로 전환**
  - 서버리스 함수(예: Cloudflare Workers / Vercel / Supabase Edge) 뒤로 키 숨김
  - 앱은 프록시 엔드포인트만 호출, 키는 서버 환경변수에만 존재
- 스토어 자산: 아이콘/스플래시, 스크린샷, 개인정보처리방침, `android.package`/`versionCode`

### 🔴 3차 — 플레이스토어 공개 배포
- `eas build -p android` (AAB) → 내부 테스트 → 비공개 테스트 → 프로덕션
- **키 프록시 전환 완료가 공개 배포의 전제 조건** (미완료 시 배포 보류)

## 12. v2 확장 백로그

> v1(기본 4탭)은 "동작하는 뼈대" 수준. 공개 배포 완성도를 위해 **습관앱 기본기 + AI 코치 차별점** 두 축으로 확장한다.
> 우선순위는 Tier 1(출시 가능선) → Tier 2(차별화) → Tier 3(성장/수익화).

### 🥇 Tier 1 — 출시 가능선 (기본기, 없으면 허접해 보임)

| 기능 | 설명 | 기술 | 데이터 변경 |
|------|------|------|-------------|
| 🔔 리마인더 알림 | 루틴별 시간 알림. 리텐션 핵심 | `expo-notifications` (로컬) | `Routine.reminderTime?` |
| 📅 유연한 반복 주기 | 매일/특정 요일/주 N회 지원 | 스케줄 계산 로직 | `Routine.schedule` (예: `{type:'daily'\|'weekly', days?:number[], perWeek?:number}`) |
| 📆 루틴 상세 + 기록 캘린더 | 항목 탭 → 월별 달성 히트맵·메모 | 기존 `records` 재활용 | (선택) `RecordEntry.memo?` |
| ⚙️ 설정 화면 + 백업/복원 | 로컬 전용이라 앱 삭제 시 데이터 증발 → JSON 내보내기/가져오기 필수 | `expo-file-system`, `expo-sharing`, `expo-document-picker` | — |
| 👋 온보딩 | 첫 실행 3~4장 안내로 이탈 방지 | 슬라이드 컴포넌트 | `Settings.onboarded` |

### 🥈 Tier 2 — 차별화 (AI 코치 = 앱 정체성)

| 기능 | 설명 |
|------|------|
| 🤖 AI 주간 리포트(자동) | 매주 일요일 "이번 주 요약 + 다음 주 목표 제안"을 Claude가 생성 |
| 💬 AI 코치 채팅 | 단발 조언 → 대화형 코칭. 멀티턴 대화 유지 |
| 🎯 정량 목표 | 완료/미완료 → 수치 기록(예: 물 1.5L/2L). `Routine.target?`, `RecordEntry.value?` |
| 🏅 배지/성취 + 최장 스트릭 | 7/30/100일 배지, 최장 스트릭 기록으로 게임화 |
| ↕️ 드래그 정렬 / 시간대 그룹 | 아침·점심·저녁 그룹, 순서 변경 |
| 📱 홈 화면 위젯 | 안드로이드 위젯 — 체크율·참여도 상승 |

### 🥉 Tier 3 — 성장 / 수익화

| 기능 | 설명 |
|------|------|
| 🌍 영어(i18n) | 한국어 전용 → 글로벌 시장 확장 |
| 💎 무료/프리미엄 | AI 기능·무제한 루틴·백업·테마를 프리미엄으로 → Claude API + 프록시 서버 비용 충당 BM |
| ☁️ 클라우드 동기화 | 기기 변경/백업 (Supabase 등) |
| 👥 친구 챌린지/공유 | 난이도 높음, 후순위. **개인 루틴은 비교 불가(같은 카테고리여도 목적·강도 제각각)** → 경쟁은 ①루틴 무관 meta 지표(최장 스트릭·달성률·완벽한 하루)로 하거나 ②옵트인 '표준 챌린지 템플릿' 안에서만. '기록 경신' 섹션이 그 토대. |

### ✅ v2 확정 범위 (스코프 고정)

전부 넣으면 출시가 무한 지연되므로 v2는 아래 핵심만:

1. 🔔 리마인더 알림 ✅ **구현 완료** (expo-notifications 로컬 알림, 루틴별 매일 알림 + 자동 예약/취소)
2. 📅 유연한 반복 주기 ✅ **구현 완료** (매일 / 특정 요일. 홈 노출·스트릭·통계·알림 전부 예정일 기준으로 동작)
3. 📆 루틴 상세 + 기록 캘린더 ✅ **구현 완료** (월별 달성 히트맵, 현재·최장 스트릭, 30일 달성률, 지난 날짜 체크 수정)
4. ⚙️ 설정 + 데이터 백업/복원 ✅ **구현 완료** (테마 선택, JSON 내보내기/가져오기(expo-file-system·sharing·document-picker), 초기화)
5. 🤖 AI 주간 리포트 ✅ **구현 완료** (이번 주 요약+다음 주 목표 생성, 주 단위 캐시. ※ "매주 자동 생성"은 백그라운드 작업 필요 → 추후)
6. 🔒 **[공개 배포 차단 항목] AI 호출 프록시 서버 전환** ✅ **코드 완료 / 배포 대기**
   - `proxy/` Cloudflare Worker (mode별 시스템 프롬프트 서버 고정, 공유 시크릿 검증)
   - 앱: `anthropic.ts`가 프록시 호출로 전환, 클라이언트 API 키 제거
   - ⏳ 남은 일(사용자): `wrangler deploy` + 시크릿 등록 + `.env`에 URL/토큰 입력 (proxy/README.md)

## 14. 클라우드 동기화 아키텍처 초안 (v3, 기기 간 연동)

> **목표**: 기기를 바꾸거나 여러 기기에서 **같은 계정으로 데이터 연속**. 오프라인에서도 전 기능 즉시 동작.
> 현재는 로컬(AsyncStorage) 전용 → **로그인 + 서버 DB + 동기화** 3요소 필요.

### 14.1 스택 선택 — Supabase
- **Auth**(이메일·소셜·매직링크) + **Postgres** + **RLS(행 수준 보안)** + Storage + (선택)Realtime 를 한 번에.
- Expo 궁합 좋고 무료 티어 넉넉. 대안: Firebase(Auth+Firestore). 직접 백엔드는 품이 큼.

### 14.2 핵심 원칙 — Local-first + Background Sync
- 로컬이 **즉각적 source of truth** → 온라인 시 서버와 **양방향 동기화**.
- 오프라인에서 전 기능 동작, 재접속 시 자동 반영. (기존 로컬 저장 구조를 버리지 않고 동기화 계층만 얹음)

### 14.3 서버 데이터 모델 (Postgres · 전 테이블 `user_id` + RLS)
```
profiles(id = auth.uid, display_name, created_at)
routines(id, user_id, title, category, icon, kind, schedule jsonb,
         reminder_time, created_at, updated_at, deleted)
records (user_id, routine_id, date, completed, updated_at)   -- PK(user_id, routine_id, date)
diary   (user_id, date, mood, text, updated_at, deleted)     -- PK(user_id, date)
settings(user_id, theme, ...)
```
- 파생 데이터(주간리포트·배지·칭호)는 로컬 계산 유지 → **서버 저장 불필요**.
- 모든 테이블에 **`updated_at` + `deleted`(tombstone)** → 동기화·삭제 전파용.

### 14.4 동기화 전략
- **변경 추적**: 레코드별 `updated_at`, 로컬에 `last_synced_at` 보관.
- **Push**: `last_synced_at` 이후 로컬 변경분을 서버로 upsert.
- **Pull**: `last_synced_at` 이후 서버 변경분을 받아 로컬 병합.
- **충돌 해결**: **Last-Write-Wins (updated_at 기준)**. (일기엔 이미 `updated_at` 존재 → 루틴/기록에 추가 필요)
- **삭제**: 물리삭제 대신 `deleted=true` tombstone → 양쪽 전파 후 주기적 정리.
- **최초 로그인 마이그레이션**: 게스트로 쌓은 로컬 데이터를 서버에 1회 업로드(중복 병합).

### 14.5 인증 흐름
- **게스트 모드 유지**: 비로그인도 로컬로 계속 사용 → 로그인 시 로컬→서버 승격.
- 진입점: 헤더 우상단 슬롯(현재 ⚙️) → **프로필/로그인 아바타로 교체**(이미 슬롯 설계됨).

### 14.6 보안 (서버 저장 시 필수 — 13장 보안 리뷰와 연결)
- **RLS 정책**: 모든 테이블 `user_id = auth.uid()` → 타인 데이터 접근 원천 차단.
- 전송 HTTPS(기본) + Supabase at-rest 암호화. **민감 필드(diary.text)는 추가 암호화 고려**.
- **PIPA/개인정보처리방침**: 일기·기분의 서버 전송·저장에 **분리 동의**(민감정보).
- **AI 프록시 강화**: 프록시가 **Supabase JWT 검증**으로 전환 → 기존 공유토큰(약한 게이트) 대체 + 사용자별 레이트리밋.

### 14.7 로컬 스키마 선행 변경
- `routines`/`records`에 **`updated_at` 추가**(현재 diary만 보유).
- 저장 계층: 동기화·용량 고려해 **AsyncStorage → expo-sqlite 이전** 검토(일기 암호화와 함께).

### 14.8 구현 단계 (별도 스프린트)
1. Supabase 프로젝트 + 스키마 + RLS 정책
2. 앱 인증(로그인/가입/로그아웃) + 게스트 모드
3. 동기화 계층(push/pull/LWW/tombstone/last_synced_at)
4. 최초 로그인 로컬→서버 마이그레이션
5. AI 프록시 JWT 검증으로 전환
6. 개인정보처리방침·동의 UI

### 14.9 확장 연결
- 로그인 기반 → **친구 비교/리더보드**(루틴 무관 meta 지표), **프리미엄(IAP)** 게이팅, 다기기 위젯.
- 비용: 텍스트 위주라 저렴 → 초기 무료 티어로 충분.
